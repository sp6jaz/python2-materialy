# Quiz W01 — Warsztat pracy analityka danych

## Instrukcja
- 5 pytań, jednokrotny wybór
- Czas: 5 minut (do wykorzystania na początku W02 jako spaced repetition)
- Odpowiedzi na końcu dokumentu

---

### Pytanie 1
Co robi polecenie `git commit -m "Opis"`?

A) Wysyła pliki na GitHub
B) Zapisuje migawkę zmian w lokalnym repozytorium
C) Pobiera pliki z internetu
D) Tworzy nowe repozytorium

---

### Pytanie 2
Który znacznik Markdown tworzy nagłówek **drugiego** poziomu?

A) `# Nagłówek`
B) `## Nagłówek`
C) `**Nagłówek**`
D) `- Nagłówek`

---

### Pytanie 3
Po co tworzymy środowisko wirtualne (`venv`) w Pythonie?

A) Żeby kod działał szybciej
B) Żeby odizolować biblioteki projektu od systemowego Pythona
C) Żeby mieć dostęp do internetu
D) Żeby szyfrować kod źródłowy

---

### Pytanie 4
Czym jest GitHub w kontekście pracy analityka danych?

A) Edytorem kodu
B) Platformą do hostowania repozytoriów Git i budowania portfolio
C) Bazą danych
D) Narzędziem do tworzenia wykresów

---

### Pytanie 5
Co to jest Mermaid?

A) Biblioteka Pythona do machine learning
B) Język do tworzenia diagramów zapisywanych jako tekst
C) System operacyjny
D) Format plików graficznych

---

## Odpowiedzi

| Pytanie | Odpowiedź | Wyjaśnienie |
|---------|-----------|-------------|
| 1 | B | `commit` zapisuje lokalnie. Do wysłania na GitHub służy `push`. |
| 2 | B | `##` = nagłówek 2. poziomu. `#` = 1. poziomu. `**` = pogrubienie. |
| 3 | B | venv izoluje zależności — każdy projekt ma własne wersje bibliotek. |
| 4 | B | GitHub to platforma do repozytoriów Git, współpracy i portfolio. |
| 5 | B | Mermaid to język diagramów tekstowych, renderowanych m.in. na GitHubie. |
